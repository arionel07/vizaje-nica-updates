# Cart Drawer — Vizaje-Nica

## Описание

Данный модуль представляет собой новую версию корзины Vizaje-Nica.

Основная цель переработки — улучшить пользовательский опыт и приблизить
интерфейс сайта к современным e-commerce решениям уровня Apple, Aesop, Rhode,
Dior Beauty и Glossier.

Вместо отдельной страницы корзины используется боковая модальная панель
(Drawer), которая открывается поверх текущей страницы и позволяет пользователю
быстро просмотреть товары, изменить количество и перейти к оформлению заказа без
лишних переходов.

---

# Причина внедрения

Старая реализация корзины требовала дополнительных переходов между страницами и
создавала лишние точки выхода пользователя.

Новая корзина решает несколько важных UX-задач:

- пользователь сразу получает подтверждение, что товар добавлен;
- уменьшается количество лишних переходов;
- увеличивается вероятность оформления заказа;
- корзина доступна из любого раздела сайта;
- ускоряется путь от выбора товара до оплаты.

# Реализовано

## Интерфейс

Создан новый Cart Drawer:

- плавное открытие справа;
- затемнение фона;
- блок товаров;
- блок итогов заказа;
- промокод;
- кнопка оформления заказа;
- кнопка продолжения покупок;
- адаптация под мобильные устройства.

---

## Товары

Для каждого товара реализовано:

- изображение;
- бренд;
- название;
- объем / характеристика;
- текущая цена;
- старая цена;
- изменение количества;
- удаление товара.

---

## Пустая корзина

Добавлено отдельное состояние Empty State.

Отображается:

- иконка корзины;
- текстовое сообщение;
- кнопка перехода в каталог.

---

## Промокоды

Реализована демонстрационная логика:

Промокод:

VIZAJE10

дает скидку:

10%

Также реализована проверка:

- промокод нельзя применить к акционным товарам;
- отображаются сообщения об успехе и ошибках.

Важно:

На данный момент проверка работает только на фронтенде и должна быть перенесена
на backend.

---

## Подсчет стоимости

Автоматически рассчитываются:

- количество товаров;
- сумма товаров;
- скидка;
- итоговая стоимость.

Пересчет происходит при:

- изменении количества;
- удалении товара;
- применении промокода.

---

## Анимации

# Что необходимо доработать

## 1. Автоматическое открытие корзины

После добавления товара корзина должна автоматически открываться.

Причина:

Пользователь должен видеть подтверждение успешного добавления товара.

Это один из самых важных UX-моментов современного интернет-магазина.

Желаемое поведение:

- пользователь нажимает "Добавить в корзину";
- товар добавляется;
- корзина автоматически открывается;
- пользователь видит добавленный товар.

---

## 2. Интеграция с реальной корзиной

На данный момент используются тестовые данные.

Необходимо подключить:

- существующую корзину сайта;
- реальные товары;
- реальные цены;
- реальные остатки.

---

## 3. Backend промокоды

Проверка промокодов должна выполняться на сервере.

Frontend должен только отображать результат проверки.

---

## 4. Синхронизация количества товаров

Количество товаров в шапке сайта должно автоматически обновляться после:

- добавления;
- удаления;
- изменения количества.

---

## 5. Сохранение корзины

Необходимо сохранить текущую логику хранения корзины сайта либо реализовать
хранение через backend/API.

---
