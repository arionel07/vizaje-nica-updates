# Vizaje-Nica UI/UX Modernization

## Описание проекта

Данная документация содержит набор независимых улучшений пользовательского интерфейса для проекта Vizaje-Nica.

Все изменения были реализованы отдельно от основной логики проекта и подготовлены таким образом, чтобы существующая команда разработки могла поэтапно внедрять их в текущую систему без полного переписывания сайта.

Главная цель проекта — повысить качество пользовательского опыта, улучшить мобильную версию сайта, увеличить конверсию оформления заказа и привести визуальный стиль сайта к современным стандартам e-commerce.

---

# Основные цели изменений

## Пользовательский опыт

Упростить взаимодействие пользователя с сайтом.

Сделать интерфейс:

* понятнее;
* быстрее;
* визуально чище;
* удобнее на мобильных устройствах.

---

## Визуальное обновление

Новый стиль ориентируется на лучшие практики:

* Apple
* Makeup
* Aesop
* Glossier
* Rhode
* Dior Beauty
* Chanel Beauty

Основные принципы:

* много свободного пространства;
* акцент на типографику;
* минимализм;
* премиальный внешний вид;
* отсутствие визуального шума.

---

## Mobile First

По данным проекта большая часть пользователей использует мобильные устройства.

Поэтому все решения разрабатывались в первую очередь для мобильной версии и только затем адаптировались под Desktop.

# Реализованные блоки

## 1. Fonts

Выполнена полная замена типографики.

Используются:

### Jost

Для:

* интерфейса;
* кнопок;
* карточек товаров;
* форм;
* административных элементов.

### Playfair Display

Для:

* заголовков;
* акцентных блоков;
* премиальной визуальной идентичности.

---

## 2. Header Top Bar

Добавлена верхняя информационная панель.

Используется для:

* бесплатной доставки;
* акций;
* специальных уведомлений;
* маркетинговых сообщений.

Полностью адаптивна.

---

## 3. Loader Screen

Создан новый экран загрузки.

Особенности:

* фирменный логотип;
* плавная анимация;
* минималистичный стиль;
* не блокирует производительность сайта.

---

## 4. Cart Drawer

Полностью переработана корзина.

Особенности:

* боковое открытие;
* затемнение фона;
* изменение количества товаров;
* удаление товаров;
* расчёт итоговой суммы;
* промокоды;
* пустое состояние корзины.

Почему это важно:

Пользователь сразу видит, что товар успешно добавлен в корзину.

Это уменьшает количество повторных нажатий и повышает конверсию.

### Важно

После интеграции необходимо автоматически открывать корзину после добавления товара.

---

## 5. Checkout

Полностью переработана страница оформления заказа.

Реализовано:

* многошаговый процесс;
* выбор доставки;
* выбор оплаты;
* подтверждение заказа;
* маска телефона;
* валидация полей;
* success modal;
* комментарий к заказу.

Подготовлены будущие возможности:

* Paynet;
* SMS подтверждение;
* подтверждение звонком;
* почтовая доставка.

---

## 6. Store Map

Полностью переработана страница магазинов.

Используется:

* MapLibre GL
* OpenFreeMap

Особенности:

* бесплатное решение;
* отсутствие API ключей;
* мобильный режим;
* кастомные маркеры;
* glass panel;
* маршруты через Google Maps.

---

# Что ещё необходимо реализовать

Данный пакет изменений не включает backend-логику.

После интеграции потребуется:

* подключение к базе данных;
* подключение корзины к реальным товарам;
* подключение оформления заказа;
* интеграция Paynet;
* сохранение комментариев;
* сохранение выбранного магазина;
* отображение новых данных в административной панели.

---

# Подключение Store Map

Для работы карты необходимо подключить библиотеку MapLibre GL.

```html
<link
	rel="stylesheet"
	href="https://unpkg.com/maplibre-gl@5.6.2/dist/maplibre-gl.css"
/>

<script src="https://unpkg.com/maplibre-gl@5.6.2/dist/maplibre-gl.js"></script>
```

Используемый стиль карты:

```text
https://tiles.openfreemap.org/styles/positron
```

Преимущества:

* бесплатно;
* отсутствуют лимиты;
* не требуется регистрация;
* отсутствует зависимость от платных сервисов.

# Итог

Данный пакет представляет собой готовый набор UI/UX улучшений для Vizaje-Nica.

Большая часть работы уже выполнена на стороне интерфейса.

Основная задача разработчиков при интеграции — подключить существующую бизнес-логику проекта и связать новые компоненты с backend-системой.
